# Stickies تحت إطار «تحسينات مقترحة» (نظام + عمل داخلي)

انسخ كل فقرة كـ sticky أصفر واحد؛ رتّب عمودًا «نظام» وعمودًا «عمل داخلي».

## نظام المنتج والبنية

1. تصفية صفوف أوامر خشب/معدن ومحتوى factory-hub بعد تعريف حقول مصنع/قسم وبناء ER يخص ذلك.
2. قرار واحد لتطبيق واجهة رئيسية؛ تحديث `docs/ARCHITECTURE.md` معه وإنهاء تعارض الوصف مع `README`.
3. سجلات طلب موحدة (`trace_id`) وبسيط عدّ JWT/403 في التطوير.
4. اختبار تكامل لـ login؛ كتابة factory-hub؛ قوائم أوامر.
5. تعميم OpenAPI/Orval لكل المسارات الموصولة فعلًا وعدم اعتماد أسلاك يدوية فقط.
6. فهارسة تدريجية ونسخ احتياطي لملف قاعدة الإنتاع غير التطوير.
7. SLA لاستيراد Excel وخطّة rollback.
8. تحقيق اتّساق المرجع ↔ جداول خشب/معدّن حيث ينطبق المحوّل؛ خطة محوّل routing المستقبلي.

## العمل الداخلي والعمليات

1. سجل ADR للقرارات الحادّة (واجهة واحدة؛ نطاق إنتاع؛ اعتماد seed مشتركة).
2. PI أو نصف رسميّة تربط `perm-scope-queries-production-hub` بمالك وتاريخ هدف.
3. RACI: ملّاك بيانات الإنتاع مقابل مالك المنصّة التقنية لقرارات السياسة.
4. دقّة صلاحيات ربع ساعة فصل رباعي؛ مراجعة المسارات التي تكتب.
5. شخص معيّن ميسّر ميرو: إطارات، عدم ازدواج، ربط لوحة مستودِع؛ لا أسرار في اللوحة.
6. كل PR يغطّى تخطيط YAML في `.cursor/plans` حيث ينطبق.
7. معيار يُعتبر «عرضًا قابلًا لتقديمه» للإدارة (مصمَّم ومجرَّب ومستخدم تجربة).

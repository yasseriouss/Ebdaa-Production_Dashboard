# مدلولات علاقات الجداول (ERD) — مرجع سريع

يُستخدم هذا الجدول عند قراءة مخطّط **Factory Hub Drizzle SQLite full ERD** على Miro أو ملفات `diagrams/data/*.mermaid`. الرموز تتبع ترميز Miro/Crow's foot المختصر في DSL.

## 1. أنواع الخط (التعريف)

| الرمز في المخطّط | الاسم التقني | المعنى |
|------------------|--------------|--------|
| `--` | identifying (متعرّف) | مفتاح الأب جزء من مفتاح الابن (نادر هنا؛ غالبًا جداول ربط مثل `auth_role_permissions`) |
| `..` | non-identifying (غير متعرّف) | الابن له مفتاح أساسي مستقل؛ الأب يُشار إليه بـ FK فقط |

## 2. الكارديناليات (من طرف إلى طرف)

| من (مصدر السهم) | الرمز | إلى (هدف السهم) | الرمز | المعنى بالعربية | مثال في المشروع |
|-----------------|-------|-----------------|-------|-----------------|-----------------|
| واحد | `one` | صفر أو كثير | `zero_or_many` | قد لا يوجد أبناء؛ وإن وُجدوا فعددهم غير محدود | مصنع → أقسام (`factories` → `departments`) |
| واحد | `one` | واحد أو كثير | `one_or_many` | يوجد ابن واحد على الأقل عند وجود الربط | أمر خشب → مراحل إنتاج |
| صفر أو واحد | `zero_or_one` | صفر أو كثير | `zero_or_many` | الربط اختياري من جهة الأب | قسم → موظفون (`department_id` اختياري) |
| واحد | `one` | صفر أو واحد | `zero_or_one` | علاقة 1:0..1 | موظف → مستخدم مصادقة (`auth_users.employee_id`) |

## 3. اختصارات أعمدة في صناديق الجداول

| الوسم | المعنى |
|-------|--------|
| **PK** | مفتاح أساسي |
| **FK** | مفتاح أجنبي يشير لجدول آخر |
| **UQ** | قيمة فريدة (مثل `order_no`, `email`) |
| **NOT_NULL** | إلزامي عند الإدراج |
| **NULL** | اختياري |
| **FK,NULL** | مفتاح أجنبي اختياري (نطاق مصنع/قسم على أمر قد يكون فارغًا) |

## 4. مجموعات الجداول (ألوان المخطّط)

| اللون / المجموعة | الجداول | دور الدومين |
|------------------|---------|-------------|
| أزرق فاتح | `factories`, `departments`, `tasks` | هيكل السعة والمهام |
| أخضر فاتح | `employees` | دليل الموظفين |
| برتقالي | `wooden_work_orders`, `wooden_production_stages` | أوامر ومراحل الخشب (معياري) |
| بنفسجي | `metal_work_orders`, `metal_production_stages`, `metal_stage_log` | أوامر ومتابعة المعدن |
| سماوي | `fh_*` | حمولة JSON للوحة / مرجع / تحليل / حفظ تلقائي |
| وردي | `shared_projects` | فهرس مشاريع (مستقل حاليًا عن مراحل الإنتاج) |
| أحمر فاتح | `auth_*` | مستخدمون، أدوار، صلاحيات، عضويات بنطاق |
| رمادي | `audit_events` | تدقيق طلبات HTTP |
| أصفر | `entity_notes` | ملاحظات متعددة الأشكال (بدون FK صريح في Drizzle) |

## 5. علاقات شائعة — جملة واحدة

| التسمية على السهم | التفسير العملي |
|-------------------|----------------|
| `has` | المصنع يضم أقسامًا |
| `owns_tasks` | القسم يملك مهام سعة |
| `employs` | المصنع يوظّف موظفين |
| `wood_stage_rows` / `metal_stage_rows` | كل أمر له صفوف مراحل إنتاج |
| `metal_daily_log_rows` | سجل يومي لكل (أمر، تاريخ، مرحلة) للمعدن |
| `fh_wood_factory` / `fh_wood_dept` | صف Hub JSON قد يُربط بمصنع/قسم للنطاق |
| `memberships` / `assignment_roles` | مستخدم ↔ دور عبر `auth_user_roles` |
| `scope_factory` / `scope_department` | نطاق صلاحية على مصنع أو قسم في العضوية |
| `audit_actor_optional` | حدث تدقيق قد يرتبط بمستخدم أو `guest` |

المصدر المعياري للأعمدة: `lib/db/src/schema/*.ts`.
